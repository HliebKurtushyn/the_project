# Інформація для команди

## Команди

### Git

#### Основні команди Git
Ініціалізувати новий репозиторій:
```
git init
```

<br>

Клонувати існуючий репозиторій до себе в директорію:
```
git clone <url>
```

<br>

Зробити пул (отримати останні зміни з віддаленого репозиторію):
```
git pull origin <branch-name>
```
_*Якщо хочете отримати останні зміни своєї гілки або master._

<br>

#### Робота з гілками
Перевірити на якій гілці ви зараз:
```
git branch
```

<br>

Створити нову гілку й одразу перемикнутися на неї (важливо):
```
git checkout -b <branch-name>
```

Перемикнутися на існуючу гілку:
```
git switch <branch-name>
```

<br>

Перейменувати поточну гілку:
```
git branch -m <new-branch-name>
```
_*Працює лише якщо ще не було пушу цієї гілки._

Якщо вже запушили гілки, але треба змінити назву, то:
```
git push origin --delete <old-branch-name>
git push origin <new-branch-name>
```

<br>

#### Коміти
Додати зміни:
```
git add <file-or-directory>
```

_Для прикладу, щоб додати всі зміни в поточній директорії:_
```
git add .
```

<br>

Зробити коміт з повідомленням:
```
git commit -m "Your commit message"
```

<br>

Переглянути статус змін:
```
git status
```
<br>

Пуш (підправка) змін на віддалений репозиторій:
```
git push origin <branch-name>
```
_*Важливо: вибирати ВИКЛЮЧНО свою гілку, а не `master`._

<br>

#### **А взагалі це все можна робити через графічний інтерфейс вашого IDE. У VS Code або PyCharm це є і дуже просто.**

<br>

### Django (manage.py)
#### Основні команди Django
Запустити локальний сервер розробки:
```
python manage.py runserver
```

<br>

Створити нову апку:
```
python manage.py startapp <app-name>
```

<br>

Створити міграції на основі змін в моделях:
```
python manage.py makemigrations
```

<br>

Застосувати міграції до бази даних:
```
python manage.py migrate
```

<br>

Зупинити локальний сервер розробки:
```
CTRL + C
```

<br>

## Початок роботи

1. Робота, само собою, почнеться з гілки master. Тож створюєте пусту папку, наприклад `<user>/Документи/Projects/the_project`, всередині
   ініціалізувати новий git репозиторій і клонувати туди проект. Ось посилання: `https://github.com/HliebKurtushyn/the_project.git`
2. Після того як репозиторій успішно клонувався, створюєте нову гілку для своєї роботи з назвою того, що ви робитимете.
   Наприклад, `feature/login-page` або `bugfix/navbar-issue`. Більше про рекомендації по назвах гілок можна прочитати в [workflow.md](./workflow.md).

### Трохи про код
- Не забувайте додавати нові залежності в `requirements.txt`
- І також, будь ласка, коли ваша гілка готова, пишіть маленьку документацію. Навіть без оформлення, щоб було більш зрозуміло
  що написано.
    - Для прикладу, якщо створили нову модель або форму, то додавайте її в документацію відразу.
- В trash.md можна залишати взагалі будь-що. Я інколи буду дивитися туди. Можете залишати різні ідеї, просто для себе нотатки. Взагалі без різниці.
- Обов'язково додавайте коментарі в код, особливо якщо щось неочевидне робите.
- Не забувайте також додавати непотрібні файли, такі як свої нотатки та інше в `.gitignore`, щоб вони не потрапляли в репозиторій.
- ВАЖЛИВО: якщо ви знайшли баг, через який не можна продовжити конкретно вашу роботу, а баг знаходиться за межами вашої гілки, тобто його зробив хтось інший, то:
    - Створити окрему гілку від вашої і master, умовно `bugfix/some-issue`, і виправити там баг.
    - Додати на гілку звідки пішов цей баг, або на master, якщо баг вже був там, issue з описом проблеми.
    - Будь ласка, не виправляйте мовчки, бо це може призвести до конфліктів з кодом і потім все може зламатись.
